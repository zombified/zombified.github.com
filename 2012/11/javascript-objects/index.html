Javascript Objects

November 19, 2012 Some notes, mostly for my own benefit, about javascript objects

Alright, I want to write down a little about javascript objects. This is mostly for my own benefit, so it may not run the full gamut of accuracy and com­plete­ness.

To start, there are really only 2 ways to create an object:

// the literal syntax
var foo = {};

// using a function to construct an object
function bar() {}
var foo = new bar();

The second method is really just shortcut syntax for something like the following:

function bar() {}
var foo = {};
bar.call(foo);

Here foo is being literally defined, and bar is being called in the context of foo. This syntax allows for more complexity than just treating bar as a con­struc­tor, as the syntax does not indicate the intent of the bar function.

Every object has a property called prototype, the prototype itself is just an object. The prototype method simply points at an object that should be used to look up properties that are not found on an object.

var foo = {};

// "bar" will not be found console.log(foo.bar);

// "foobar" will be printed to the console because the object "foo" does not // contain the property "bar", but it's prototype does. foo.prototype.bar = "foobar"; console.log(foo.bar);

// "eggs" will be printed because the object does actually have the property "bar" foo.bar = "eggs"; foo.prototype.bar = "bacon"; console.log(foo.bar);
