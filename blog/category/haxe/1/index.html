Experiments with haXe

I'm interested in playing around with haXe -- it seems like a concept I could get behind and have some fun with.

In trying to get a working little example that uses webgl/canvas, I've come accross a file called "Html5Dom.hx".

I was curious about the origin of this file, so I did a small amount of looking. The source seems to be from the Jeash project, which was created from the merging of the Canvas-NME and Neash projects. However, the Html5Dom.hx file seems to not contain any reference to webgl until changeset 27 (a0f27fc5c734) on August 16th 2010. At this point a port of the Khronos groups WebGL IDL was added to the file. Further changes fixed areas of the initial port by adding support for the TypedArray ID from the Khronos group.

Easy enough information to find out, but is the file relevant (the webgl spec is moving fast, or so I've read), or is there some other work out there to create an extern library for haxe that enables the use of WebGL for the javascript target?

As far as I can tell, this is the best, and possibly only, extern library made publicly available for WebGL use in haXe. I found a blog post by a 'Michael Cann' that uses a version of the Html5Dom.hx extern library for a test project that produces 5,000,000 point sprites on screen, so I think the library is still relevant.

I did find the hx.html5 project, but that seems to just contain extern's for the 2D canvas context.

I also found a extern library for three.js, called three.js-completion. This library doesn't seem to be updated to the latest three.js version though. Might still be a starting point.
