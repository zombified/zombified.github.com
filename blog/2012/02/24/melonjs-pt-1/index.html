MelonJS pt 1

Last December I participated in the Ludum Dare (#22), and ultimately used melonJS as my framework of choice.

The next Ludum Dare is coming up, and this time it's the 10th anniversary of the competition!

I need to be prepared.

Therefore, I have decided to do a short series, for my benefit, on how to make a melonJS game.

The game I will be developing is going to be called Minefield, and is based off of the Doger game from Invent With Python.

The full source code will be found on github under my minefield project. Specifically, the code in this post can be found at this commit.

All required files to make this app work are stored in the repository. The libs directory contains all libraries used that aren't specifically created for this project.

There are only 2 significant files (outside of melonJS) in the repo so far:

  1. index.html
  2. js/PlayScreen.js

index.html

This is where most of the magic happens for the base structure needed for the game.

Check out the source, the code is fairly small, and all the important sections have comments.

The one line that I'd like to make a note about is:

<div id="app"></div>

It provides a location for melonJS to create it's render surface. The ID is arbitrary, and is used in the call to "me.video.init".

PlayScreen.js

For now, this is just an empty object. We will be using this object to build up our PLAY state later on.
